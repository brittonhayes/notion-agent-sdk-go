# Contributing

Contributions are welcome. This project is released into the public domain under the [Unlicense](UNLICENSE), so your contributions will be too.

## Getting started

1. Fork the repository
2. Clone your fork
3. Create a feature branch (`git checkout -b my-feature`)
4. Make your changes
5. Run checks: `go vet ./...`
6. Commit your changes (`git commit -m "Add my feature"`)
7. Push to your branch (`git push origin my-feature`)
8. Open a pull request

## Guidelines

- Use Go standard library only. No external dependencies.
- Follow existing code style and conventions.
- Add doc comments for all exported types, functions, and methods.
- Keep changes focused. One feature or fix per pull request.

## Reporting issues

Open an issue on GitHub with a clear description of the problem, steps to reproduce, and your Go version.
